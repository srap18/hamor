## التغييرات

### 1. قاعدة البيانات
- إضافة عمود `elite_vip_level` في `profiles` (افتراضي 0، حد أقصى 5)
- **تصفير جميع اللاعبين** (`vip_level = 0` للنظام القديم + `elite_vip_level = 0`)
- جدول `elite_vip_tier_config` (إعدادات قابلة للتعديل من الأدمن)
- حذف جداول `vip_daily_claims`, `vip_shield_claims`, `royal_box_claims` (نظام قديم)
- RLS: العمود `elite_vip_level` للقراءة فقط من العميل، الكتابة عبر service_role فقط

### 2. منتجات Paddle (5 اشتراكات شهرية)
- `elite_vip_1_monthly` … `elite_vip_5_monthly`
- ربط webhook ليُحدّث `elite_vip_level` تلقائياً عند الاشتراك/الإلغاء

### 3. أمان الخادم (Server-side)
- وظيفة `get_effective_combat_stats(user_id)` تطبّق بونص الهجوم/الدفاع من DB فقط
- وظيفة `get_effective_shop_price(item_id, user_id)` تطبّق الخصم من DB فقط
- جميع الـ Edge Functions الموجودة (`record-attack`, `shop purchases`, إلخ) تُحدّث لاستخدام هذه الدوال
- **يستحيل** على العميل تعديل المستوى — السيرفر هو المصدر الوحيد

### 4. المرئيات (5 شارات/إطارات)
- توليد 5 شارات عالية الجودة مستوحاة من الصورة (مرساة، درع، تاج+جمجمة، سفينة، تنين)
- مكوّن `EliteVipBadge` يظهر بجانب الاسم في الشات والملف الشخصي
- مكوّن `EliteVipFrame` يحيط بصورة الأفاتار

### 5. ميزات حسب المستوى
- **مستوى 3+**: تراكب دخول عالمي (Banner بث في الوقت الفعلي عبر Supabase Realtime، شفاف بملء الشاشة لـ5 ثوانٍ يعرض الاسم والشارة عند تسجيل الدخول)
- **مستوى 4**: اسم بلون ذهبي في الشات والمكونات العامة
- **مستوى 5**: اسم بتدرج ذهبي مرصع + توهج

### 6. الواجهة
- إعادة كتابة صفحة `/vip` لعرض المستويات الـ5 مع زر اشتراك Paddle لكل مستوى
- شارة Elite في الـ HUD والشات وقوائم اللاعبين والملف الشخصي
- لوحة الأدمن: إضافة قسم لتعديل البونصات والأسعار

### 7. إزالة النظام القديم
- حذف `vip-perks.ts`, `vip_daily_claims`, `vip_shield_claims`
- إزالة المنطق المرتبط بـ `vip_level` القديم في الكود
- إبقاء عمود `vip_level` في DB مؤقتاً (لتجنب كسر شيء) لكن لا يُستخدم

## الأمور التقنية (تفاصيل المنفّذ)
- Webhook Paddle يربط `price_id` (`elite_vip_X_monthly`) → `elite_vip_level = X`
- عند `subscription.canceled` أو انتهاء الفترة → `elite_vip_level = 0`
- الـ overlay العالمي يستخدم channel `elite-login-broadcasts` يُرسل event من الـbackend عند تسجيل دخول لاعب VIP 3+
- جميع حسابات الهجوم/الدفاع/الأسعار تُجرى **حصرياً** في server functions أو DB functions
- لا توجد دوال RPC تسمح للعميل بتمرير `vip_level` كمدخل

هل توافق على هذا التنفيذ بالكامل بهذه الأسعار والأسماء والمكافآت؟ أم تريد تعديل شيء قبل أن أبدأ؟
