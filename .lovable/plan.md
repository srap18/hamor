## الهدف
- السفينة تحتفظ بسعتها كاملة (مثلاً 1,000,000 سمكة) بدون أي تعديل.
- الكمية المصطادة = `السعة × (الثواني المنقضية ÷ مدة الرحلة)`.
- إيقاف الصيد في أي لحظة يعطي نتيجة فورية حتى لو كانت الكمية ضخمة.
- لا يوجد أي سقف داخلي (لا 5000 ولا غيره).

## المشكلة الحالية
الدالة `collect_fishing_reward` تُدخل صفًا واحدًا في جدول `fish_stock` لكل سمكة عبر `generate_series(1, _qty)`. لما الكمية تصير 300,000 أو مليون، الإدخال يأخذ وقت طويل ويسبب timeout — ولهذا أضفنا سقف 5000 سابقًا.

## الحل
نخزّن السمك المصطاد كصف واحد مجمَّع يحمل عمود `quantity` بدل صف لكل سمكة. هكذا الإدخال يكون فوريًا مهما كانت الكمية.

## الخطوات (migration واحدة)

### 1) تعديل جدول `fish_stock`
- إضافة عمود `quantity integer NOT NULL DEFAULT 1`.
- إضافة فهرس على `(user_id, fish_id)` إن لم يكن موجودًا.

### 2) تحديث `collect_fishing_reward`
- إزالة `_stock_cap` وكل السطور المتعلقة بالسقف.
- استبدال:
  ```sql
  INSERT INTO fish_stock(...) SELECT ... FROM generate_series(1, _qty);
  ```
  بصف واحد:
  ```sql
  INSERT INTO fish_stock(user_id, fish_id, ship_id, caught_at, base_value, quantity)
  VALUES (_uid, _chosen, _ship_id, now(), _unit_value, _qty);
  ```
- إبقاء معادلة الحساب كما هي:
  ```
  _ratio := LEAST(1, _elapsed / _duration);
  _qty   := FLOOR(_capacity * _ratio) * _luck_mult;
  ```

### 3) تحديث `get_fish_stock_summary`
- تغيير `COUNT(*)` إلى `SUM(quantity)` للمجاميع.
- تغيير `SUM(base_value)` إلى `SUM(base_value * quantity)` للقيمة.

### 4) تحديث `sell_fish`
- ضرب القيمة في `quantity` عند احتساب الذهب.
- حذف الصفوف المختارة كما هي (الصف الواحد يمثل كل الكمية المصطادة من تلك الرحلة).

### 5) تحديث `feed_daughter`
- استخدام `quantity` بدل عدّ الصفوف عند خصم السمك.

## الأمثلة (بعد التعديل)
| السعة | المدة الكاملة | لحظة الإيقاف | المصطاد |
|---|---|---|---|
| 1,000,000 | 1,000,000 ث | 100 ث | 100 |
| 1,000,000 | 1,000,000 ث | 1,000,000 ث | 1,000,000 |
| 300,000 | 2700 ث | 10 ث | 1,111 |
| 300,000 | 2700 ث | 2700 ث | 300,000 |

## الملفات المتأثرة
- `supabase/migrations/<جديد>.sql` — كل تغييرات قاعدة البيانات.
- لا تغيير مطلوب على الفرونت إند: الواجهات تستخدم `get_fish_stock_summary` و`sell_fish` وستعمل كما هي بعد تحديث الدوال.

## النتيجة
- إيقاف أي سفينة في أي ثانية يعطي نتيجة فورية.
- لا يوجد أي حد أقصى — السعة المعلنة هي الحد الحقيقي.
- الأداء ثابت سواء صادت 10 أو مليون سمكة (صف واحد فقط في `fish_stock` لكل رحلة).
